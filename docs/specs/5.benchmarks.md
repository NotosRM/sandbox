## Часть 3: Бенчмарки производительности

### Зачем

Сравнение технологий — это не только «что удобнее», но и «что быстрее». Когда ты сравниваешь zustand vs reatom на рендеринге 10 000 элементов, тебе нужны цифры, а не ощущения.

Три уровня измерений:

- **Микробенчмарки** — чистая логика (как быстро редьюсер обрабатывает действие)
- **Рендер-бенчмарки** — измерение времени рендеринга компонентов
- **Bundle size** — размер итогового бандла

### Подход

Утилиты для замеров — в `shared/`. Микробенчмарки — через `vitest bench`. Bundle size — через готовый скрипт в `scripts/`.

### Структура в `shared/`

```
shared/src/
├── benchmark/
│   ├── index.ts
│   ├── measure.ts            # обёртка над performance.mark/measure
│   ├── react-profiler.ts     # компонент-обёртка над React.Profiler
│   └── stress.ts             # генераторы данных (10k items, deep tree, и т.д.)
```

### Микробенчмарки через Vitest

Vitest поддерживает bench-режим из коробки:

```typescript
// experiments/zustand-vs-reatom-a/src/store.bench.ts
import { bench, describe } from 'vitest';
import { createStore } from './store';

describe('store updates', () => {
  bench('1000 sequential updates', () => {
    const store = createStore();
    for (let i = 0; i < 1000; i++) {
      store.increment();
    }
  });
});
```

Запуск: `pnpm bench` (новый скрипт в шаблонах: `"bench": "vitest bench"`).

### Bundle size analysis

Скрипт `scripts/bundle-size.ts`:

```bash
pnpm run bundle-size <имя_эксперимента>
```

Логика:

1. Собирает эксперимент: `cd experiments/<имя> && pnpm build`
2. Анализирует `dist/` — суммирует размеры JS/CSS, gzip, brotli
3. Выводит таблицу: файл, raw, gzip, brotli
4. Опционально — сравнивает с предыдущим прогоном (сохраняет в `.bundle-size-history.json`)

Зависимости в корне:

```jsonc
{
  "devDependencies": {
    "rollup-plugin-visualizer": "^5.x",
    "gzip-size": "^7.x",
    "brotli-size": "^4.x",
  },
}
```

### React Profiler обёртка

В `shared/benchmark/react-profiler.ts` — компонент-обёртка, которая логирует время рендеринга в console.table:

```tsx
import { Profiled } from '@sandbox/shared/benchmark';

<Profiled id="counter-list">
  <CounterList items={items} />
</Profiled>;
```

После размонтирования выводит сводку: количество рендеров, среднее/мин/макс время.

### Команда Claude Code

`.claude/commands/experiment-add-bench.md` — добавляет в эксперимент `*.bench.ts` файл с базовым бенчмарком, обновляет `package.json` (скрипт `bench`).

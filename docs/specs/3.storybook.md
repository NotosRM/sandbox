# Frontend Sandbox — Дополнение: Production-grade инфраструктура

Третье дополнение к основной спецификации. Описывает инфраструктуру, которая делает песочницу пригодной для исследования продакшен-сценариев: Storybook, MSW, бенчмарки, CI/CD и ADR.

---

## Часть 1: Storybook

### Зачем

Когда эксперимент про UI-компонент (кнопка с анимацией, сложный инпут, дизайн-система), тебе нужно:

- Видеть компонент изолированно, без всего приложения
- Прокликать все его состояния (loading, error, disabled, hover)
- Документировать API через args и controls
- В будущем добавить visual regression тесты

### Подход

Storybook — это не общая инфраструктура, а отдельный шаблон `react-storybook`. Не каждый эксперимент про UI, поэтому нет смысла тащить его во все React-эксперименты.

### Новый шаблон: `react-storybook`

```
templates/react-storybook/
├── package.json.tpl
├── tsconfig.json
├── vite.config.ts
├── index.html
├── .storybook/
│   ├── main.ts
│   └── preview.ts
├── src/
│   ├── main.tsx
│   ├── App.tsx
│   ├── Button.tsx
│   ├── Button.stories.tsx       # пример story
│   └── Button.test.tsx
└── CLAUDE.md
```

### Зависимости (только в этом шаблоне, не в корне)

```jsonc
{
  "devDependencies": {
    "storybook": "^8.x",
    "@storybook/react-vite": "^8.x",
    "@storybook/test": "^8.x",
    "@storybook/addon-essentials": "^8.x",
  },
}
```

### Скрипты

```jsonc
{
  "scripts": {
    "dev": "vite",
    "storybook": "storybook dev -p 6006",
    "build-storybook": "storybook build",
    "test": "vitest",
    "test:run": "vitest run",
  },
}
```

### Конвенция расположения stories

Stories лежат рядом с компонентом, как и тесты:

```
src/
├── Button.tsx
├── Button.stories.tsx     # stories
├── Button.test.tsx        # unit-тесты
└── ...
```

### Visual regression (опционально)

Для визуального тестирования — встроенный Storybook test-runner на базе Playwright. Не включается по умолчанию, добавляется через команду `/experiment:add-visual-tests` (см. ниже).

### Команда Claude Code

`.claude/commands/experiment-add-visual-tests.md` — добавляет в эксперимент с Storybook визуальные тесты через `@storybook/test-runner` и Playwright.

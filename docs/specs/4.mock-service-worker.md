## Часть 2: MSW (Mock Service Worker)

### Зачем

Любой продакшен-сценарий — это работа с API. Авторизация, пагинация, оптимистичные обновления, обработка ошибок, race conditions, retry-логика, кэширование. Это невозможно осмысленно тестировать без мока сетевого слоя.

MSW перехватывает HTTP-запросы на уровне Service Worker (в браузере) или через node-интерсептор (в тестах). Один и тот же набор моков работает и в dev-сервере, и в Vitest, и в Playwright.

### Подход

MSW — общая инфраструктура. Лежит в `shared/`, переиспользуется любым экспериментом одной строкой импорта.

### Структура в `shared/`

```
shared/
├── package.json
├── src/
│   ├── index.ts
│   ├── measure.ts
│   ├── logger.ts
│   ├── types.ts
│   └── msw/
│       ├── index.ts              # экспорт setupMocks()
│       ├── handlers.ts           # базовые handler'ы для типовых сценариев
│       ├── browser.ts            # setupWorker для браузера
│       ├── node.ts               # setupServer для тестов
│       └── scenarios/            # готовые сценарии для переиспользования
│           ├── auth.ts           # логин, токены, refresh, 401
│           ├── pagination.ts     # cursor + offset пагинация
│           ├── errors.ts         # 500, 503, network failure, timeout
│           └── slow-network.ts   # симуляция медленного API
```

### Зависимости в корневом `package.json`

```jsonc
{
  "devDependencies": {
    "msw": "^2.x",
  },
}
```

### Использование в эксперименте

```typescript
// experiments/auth-flow/src/main.tsx
import { setupMocks } from '@sandbox/shared/msw';
import { authScenario } from '@sandbox/shared/msw/scenarios/auth';

if (import.meta.env.DEV) {
  await setupMocks(authScenario);
}
```

В тестах:

```typescript
import { setupServer } from '@sandbox/shared/msw/node';
import { authScenario } from '@sandbox/shared/msw/scenarios/auth';

const server = setupServer(...authScenario);
beforeAll(() => server.listen());
afterAll(() => server.close());
```

### Команда Claude Code

`.claude/commands/experiment-add-msw.md` — подключает MSW к существующему эксперименту, создаёт `src/mocks.ts` с базовыми handler'ами и инициализирует в `main.tsx`.

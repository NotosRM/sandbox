# Frontend Sandbox

Монорепо-песочница для фронтенд-экспериментов. pnpm workspaces.

## Структура

- `templates/` — шаблоны для новых экспериментов (не воркспейсы, копируются как есть)
- `experiments/` — все эксперименты (каждый — отдельный pnpm workspace)
- `shared/` — переиспользуемые утилиты (`@sandbox/shared`)
- `scripts/` — CLI: create, list, clean

## Создание эксперимента

```bash
pnpm run create -- --template <шаблон> --name <имя>
pnpm install   # обязательно после создания — добавляет новый workspace
```

Доступные шаблоны: `react-vite`, `vue-vite`, `custom-build`, `react-full`, `vue-full`

## Команды

| Команда                                    | Описание                           |
| ------------------------------------------ | ---------------------------------- |
| `pnpm run create -- --template X --name Y` | Создать эксперимент                |
| `pnpm run list`                            | Список всех экспериментов          |
| `pnpm run clean <name>`                    | Удалить dist/.vite эксперимента    |
| `pnpm test`                                | Vitest watch по всем экспериментам |
| `pnpm test:run`                            | Однократный прогон всех тестов     |
| `pnpm lint`                                | Oxlint                             |
| `pnpm format`                              | Prettier                           |

## Конвенции

- Имена экспериментов: `kebab-case`
- Сравнения двух технологий: `<tech1>-vs-<tech2>` (два отдельных эксперимента `...-a` и `...-b`)
- Тесты: co-located, `*.test.ts(x)` рядом с кодом
- E2E: в `e2e/*.spec.ts`, только если нужен реальный браузер
- Каждый эксперимент имеет `README.md` с полями: шаблон, дата, статус, цель, выводы

## Стек

- TypeScript strict
- Oxlint (линтинг)
- Prettier (форматирование)
- Vitest (unit/integration тесты)
- Playwright (E2E — добавляется в эксперимент по необходимости)

## Правила

- Не модифицируй файлы в `templates/` без явного запроса
- Не выноси код в `shared/` пока он не используется в 3+ экспериментах
- Каждый эксперимент должен запускаться независимо через `pnpm dev`
- После создания эксперимента всегда запускай `pnpm install` из корня

## Часть 4: CI/CD (GitHub Actions)

### Зачем

Без CI ты однажды сломаешь один эксперимент, чиня другой, и не заметишь. Минимальный workflow, который при каждом пуше прогоняет линтер, типчекер и тесты по всем экспериментам.

### Workflow: `.github/workflows/ci.yml`

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      - name: Typecheck
        run: pnpm -r typecheck

      - name: Test
        run: pnpm test:run
```

### Что важно понимать

Песочница — это не продакшен. CI здесь не блокирует деплой, потому что деплоя нет. Его задача — давать сигнал: «эксперимент X сломался при изменениях в shared/» или «тест Y упал после обновления зависимости». Не более того.

### Автоматический бенчмарк (опционально)

Второй workflow `.github/workflows/bench.yml` — запускается вручную через `workflow_dispatch`, прогоняет все `*.bench.ts` и публикует результат как комментарий к коммиту. Не нужно по умолчанию, добавляется когда появятся первые серьёзные сравнения производительности.

### Команда Claude Code

`.claude/commands/experiment-skip-ci.md` — помечает эксперимент как `archived`, добавляет его в `.ciignore` (кастомный файл, читается workflow), CI перестаёт его проверять. Полезно для сломанных или брошенных экспериментов.
